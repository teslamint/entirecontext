## Theme

Graduate all 8 carry-forward items from the v0.11.0 retrospective. Code changes allowed only when diagnosis warrants them, gated by hard caps. No new features.

## Approach: Option F — Carry-Forward Triage

Each carry-forward is triaged into one of three categories:

1. **Immediate** — docs/process work, no code required
2. **Data check** — record current values, pass/fail judgment
3. **Diagnose-then-decide** — code changes permitted within hard caps

The `sessions_with_application` dashboard query (line 222-233) requires `ca.retrieval_selection_id IS NOT NULL` JOIN, which is correct — only counting applications linked to a retrieval selection.

**Decision: won't-fix the code.** The formula is correct. The rate is low because the loop is not being exercised — same root cause as lesson_reuse_rate=0%. Intervene improvement requires dogfooding volume, not measurement changes.

3. **Time box**: Day 4 cutoff. Unresolved items get won't-fix + rationale.
