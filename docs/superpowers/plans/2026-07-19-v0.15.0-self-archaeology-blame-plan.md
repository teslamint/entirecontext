## Units

### U1 — `confirm_candidates_batch()` (core)

**File**: `src/entirecontext/core/decision_candidates.py`

**Signature**:

```python
def confirm_candidates_batch(
    conn,
    *,
    source_type: str | None = None,
    min_confidence: float = 0.0,
    dry_run: bool = False,
    reviewer: str = "cli",
    repo_path: str | None = None,
    page_size: int = 200,
) -> dict[str, Any]:
```

**Dry-run path** (no mutation): direct SQL over ALL matching pending rows — no LIMIT:
`SELECT confidence FROM decision_candidates WHERE review_status='pending'` + optional `AND source_type=?`. Bucket confidences by 0.1 ([0.0,0.1) … [0.9,1.0]) in Python. Return `{"dry_run": True, "total_pending": int, "eligible": int, "distribution": {"0.9-1.0": n, ...}}` where `eligible` counts `confidence >= min_confidence`.

**Confirm path**: loop `list_candidates(conn, status="pending", source_type=source_type, min_confidence=min_confidence, limit=page_size)` until no unattempted rows remain. **Loop-termination guard**: `confirm_candidate`'s failure path rolls a candidate back to `pending`, so a failed candidate would reappear forever — keep an `attempted: set[str]` of candidate IDs, skip already-attempted rows, and break when a page contains only attempted IDs. Each candidate: `confirm_candidate(conn, cid, reviewer=reviewer, repo_path=None)` — `repo_path=None` skips the per-candidate embedding pass (decision_candidates.py:245-255). Catch `ValueError` (lost race / already-processed) and `Exception` per candidate into `failed`; never abort the batch. After the loop, if any confirmed and `repo_path` given: replicate the existing tail of `confirm_candidate` — load config, `decisions.auto_embed` gate, `generate_embeddings(conn, repo_path, decisions_only=True)` once, whole block in try/except pass. Also compute `skipped_below_threshold` = pending-matching-source count minus eligible (same no-LIMIT SQL as dry-run). Return `{"confirmed": [decision_ids], "failed": [candidate_ids], "skipped_below_threshold": int}`.

**Test scenarios** (`tests/test_decision_candidates_batch.py`, seed via `persist_candidate` helper style from test_confirm_commit_link.py):
- Happy: 3 archaeology candidates ≥ threshold → all promoted; each has a `decision_commits` row (source_id is a 40-hex SHA). Covers S3.
- Happy: dry-run returns bucket counts and `review_status` unchanged for every row. Covers S2.
- Edge: threshold boundary — candidate with confidence exactly equal to `min_confidence` is confirmed.
- Edge: source filter — assessment/checkpoint pending candidates present (mirror live DB: 2+7) are untouched when `source_type="archaeology"`.
- Edge: >page_size pending candidates (seed page_size=5, 12 candidates) → all 12 promoted (pagination past the `limit` default).
- Error: monkeypatch `create_decision` to raise for one candidate → that candidate back to `pending`, others promoted, its ID in `failed`, loop terminates.
- Error: already-confirmed candidate in the ID set → `ValueError` swallowed into `failed`, batch continues.
- Integration: embeddings — monkeypatch `generate_embeddings`; assert called exactly once for a 3-candidate batch when auto_embed on, zero times when `repo_path=None` or dry-run.

### U3 — `core/blame_decisions.py` (new)

**Public API**:

```python
@dataclass
class BlameAnnotation:
    commit_sha: str
    line_ranges: list[tuple[int, int]]
    decision_id: str
    title: str
    rationale_excerpt: str          # first 200 chars of rationale, "" if None
    rejected_count: int             # len(json rejected_alternatives)
    staleness_status: str           # fresh|stale|superseded|contradicted

def annotate_file(
    conn, repo_path: str, file: str,
    start_line: int | None = None, end_line: int | None = None,
) -> dict[str, Any]:
    # returns {"annotations": list[BlameAnnotation],
    #          "uncommitted_ranges": list[tuple[int, int]],
    #          "annotated_sha_count": int, "total_sha_count": int}
```

**Mechanics**: `subprocess.run(["git", "blame", "--porcelain", *(-L args if given), "--", file], cwd=repo_path, capture_output=True, text=True, timeout=30)` following the git_utils.py pattern (catch `TimeoutExpired`/`FileNotFoundError`; non-zero returncode → raise `ValueError(stderr.strip())` — covers untracked/binary/missing file). Parse porcelain headers with `re.match(r"^([0-9a-f]{40}) (\d+) (\d+)(?: (\d+))?$", line)` — group 1 SHA, group 3 final line number; tab-prefixed lines are content, other header keys skipped. Build `final_line → sha` map; collapse consecutive lines per SHA into ranges. `"0"*40` SHAs go to `uncommitted_ranges` and are excluded from the query. Single query on the deduplicated SHA set:

```sql
SELECT dc.commit_sha, d.id, d.title, d.rationale, d.rejected_alternatives, d.staleness_status
FROM decision_commits dc JOIN decisions d ON d.id = dc.decision_id
WHERE dc.commit_sha IN (?, ...)
```

One `BlameAnnotation` per (commit_sha, decision) pair — a SHA can carry multiple decisions and a decision multiple SHAs.

**Test scenarios** (`tests/test_blame_decisions.py`, real git repo via `git_repo`/`ec_repo` fixtures per conftest):
- Happy: repo with 2 commits, decision linked to commit 1 via `link_decision_to_commit` → annotation carries title/excerpt/ranges for commit-1 lines only. Covers S4.
- Happy: two decisions linked to the same SHA → two annotations.
- Edge: file whose SHAs have no `decision_commits` rows → empty annotations, `total_sha_count` > 0.
- Edge: uncommitted working-tree modification → all-zeros SHA lands in `uncommitted_ranges`, not in the query.
- Edge: `start_line`/`end_line` restrict blame to the range (`-L` passed to git), SHAs outside the range absent.
- Edge: rationale None → excerpt `""`; rationale > 200 chars → truncated to 200.
- Error: untracked file → `ValueError` with git's stderr.
- Happy: `staleness_status` pass-through for a `superseded` decision. Covers S5.

### U4 — `ec blame --decisions` rendering

**File**: `src/entirecontext/cli/blame_cmds.py`

Add `decisions: bool = typer.Option(False, "--decisions", help="Annotate with decision history")` to `blame_cmd`. Guard first: `if decisions and summary:` → `console.print("[red]--summary and --decisions are mutually exclusive.[/red]")`, `raise typer.Exit(1)` (the summary path returns early at :52-66 and has no annotation surface). When `decisions` and not summary: after the existing attribution table (existing output unchanged byte-for-byte when flag absent), call `annotate_file(conn, repo_path, file, start_line, end_line)` **before** the `finally: conn.close()` boundary (move the annotate call inside the same try block that fetches attributions). - Header `Decision annotations: <file>`.
- Per annotation: `  <sha[:8]> lines <s>-<e>[, <s>-<e>…] — <title>` with `[STALE:<status>]` suffix in yellow when `staleness_status != "fresh"`, then dim excerpt line, then `(N rejected alternatives)` when N > 0, then for stale ones: `  ↳ re-verify: ec decision get <decision_id>`.
- `uncommitted_ranges` → `  lines <s>-<e>: uncommitted (no blame history yet)`.
- No annotations at all → `[dim]No recorded decisions for this file's commits — absence of links, not evidence that no decisions were made.[/dim]`.
- `ValueError` from `annotate_file` → red message, exit 1.

### U5 — Operational bootstrap (non-code)

Runbook, executed after U1–U4 merge readiness (order matters: batch confirm must exist before candidates pile up):

1. `uv run ec archaeologize --dry-run` — record pending count.
2. Repeat `uv run ec archaeologize --pr-bodies --limit 100` until a following `--dry-run` reports 0 patch extractions and 0 PR enrichments pending (resumable via `archaeology_processed`; retryable enrichment shipped v0.14.0).
3. `uv run ec decision candidates confirm-batch --source archaeology --min-confidence 0.99 --dry-run` — inspect the distribution (the 0.99 value is irrelevant in dry-run; the distribution covers all pending).
4. Choose the real threshold from the distribution (spec Open Decision — operator judgment; record the chosen value and rationale in CHANGELOG/release notes).
5. `uv run ec decision candidates confirm-batch --source archaeology --min-confidence <chosen>`.
6. Record before/after metrics (the two coverage queries from the spec's Assumptions) — evidence for Success Criteria 1–2.
7. `uv run ec blame --decisions <a file named in a confirmed decision>` — paste output; evidence for Success Criterion 3.

**Verification evidence**: transcript + metric numbers stored under `.release-loop/evidence/U5/` and summarized in the retro. Covers S1, S2, S3 end-to-end on live data.

## Mutation/failure-state matrix

Stateful ceremony present (batch confirm mutates `decision_candidates`, `decisions`, join tables, `embeddings`; operational run mutates `archaeology_processed`). Deviation authority: `docs/solutions/workflow-issues/review-introduced-state-machine-deviation.md` (path per skill contract; directory not present in this repo — flagged, not silently dropped).

| Transition | Pre-state | Action | Post-state | Unit / evidence owner |
|---|---|---|---|---|
| T1 candidate claim+promotion (per batch item; reuses `confirm_candidate` ceremony unchanged) | `review_status='pending'` | CAS UPDATE → BEGIN IMMEDIATE (decision + joins + back-pointer) | `confirmed` + `promoted_decision_id` + decision/`decision_commits` rows | U1 / `.release-loop/evidence/U1/` |
| T2 batch-end embedding | no vectors for new decisions | `generate_embeddings(decisions_only=True)` once | embedding rows present | U1 / `.release-loop/evidence/U1/` |
| T3 archaeology processing marks | commit absent from `archaeology_processed` | `ec archaeologize` batch run | row present; enrichment flags per v0.14.0 state model | U5 / `.release-loop/evidence/U5/` |

Outcome classes:

- **T1** — success: batch test promotes all eligible. Forced failure: monkeypatch `create_decision` raise on one candidate (safe injection — test process only, isolated `ec_db` fixture); expect claim compensated back to `pending` (existing outer-except, decision_candidates.py:230-243), batch continues. Rerun: re-invoking confirm-batch skips confirmed rows (pending filter) and retries previously failed ones — idempotent convergence. Rollback/compensation: the claim-rollback IS the compensation; the BEGIN IMMEDIATE block rolls back decision+joins atomically. Headless: CLI is non-interactive; no TTY dependency (CliRunner tests prove it). Cancellation/abort: KeyboardInterrupt between candidates leaves only whole-candidate states (each candidate is atomic); resume = rerun. Abort *inside* one candidate's promotion → SQLite rolls back the open transaction; worst case is the documented pre-existing "claim committed, promotion not started" stuck state (confirm_candidate docstring), unchanged by this plan.
- **T2** — success: called-once assertion. Forced failure: monkeypatch `generate_embeddings` raise → swallowed (mirrors existing per-candidate try/except), confirmations stand. Rerun: regeneration is safe (derived data). Rollback: not applicable — embeddings are derived, regenerable data with no consumer that assumes their absence; no compensation needed. Headless: config-gated (`decisions.auto_embed`), no interaction. Cancellation: abort during embed leaves confirmed decisions intact; embeddings regenerate on next confirm or `ec index` run.
- **T3** — transitions owned by already-shipped v0.13.1/v0.14.0 code, not modified by this plan. Success/rerun/failure/cancellation evidence: v0.14.0 regression suites (streaming lifecycle, retryable enrichment, circuit recovery — see ROADMAP v0.14.0 section) plus U5's live dry-run convergence to 0 pending. Rollback: not applicable — `archaeology_processed` marks are append-only idempotency records; "undo" is meaningless (re-processing is prevented, not caused, by their presence). Headless: CLI batch runs are non-interactive. Forced failure at planning scope: not re-injected here — no new code path exists to inject into; injection would test shipped code already covered by its own suite (concrete reason for not-applicable).
